# 📖 Glossary — قاموس المشروع

> يمنع الفوضى: كل مطور ووكيل AI يستخدم نفس الكلمة بنفس المعنى.

| المصطلح | الإنجليزية | التعريف |
|---------|-----------|---------|
| **مصدر** | Source | مؤسسة رسمية تصدر معلومات أولية (Fed, SEC, BLS) |
| **وثيقة** | Document | الملف الخام من المصدر (PDF, HTML, RSS) |
| **حقيقة** | Fact | معلومة مستخرجة مباشرة من الوثيقة دون تفسير |
| **حدث** | Event | حدث مالي مُشتق من Fact أو أكثر (قرار فائدة، أرباح) |
| **دليل** | Evidence | الإثبات الذي يربط الـ Fact أو Event بالوثيقة الأصلية |
| **رؤية** | Insight | استنتاج ينتجه النظام اعتماداً على الأدلة |
| **منشور** | Publication | المحتوى النهائي المنشور (خبر، تقرير، تنبيه) |
| **محول** | Adapter | مكون يعرف كيف يتعامل مع نوع مصدر محدد |
| **جالب** | Fetcher | يجلب الوثيقة من المصدر (HTTP, RSS) |
| **محلل** | Parser | يحلل بنية الوثيقة (HTML structure, PDF layout) |
| **مستخرج** | Extractor | يستخرج النص/الحقائق من الوثيقة |
| **مصنف** | Classifier | يصنف الحدث في Taxonomy |
| **محرر** | Editor | يراجع ويحرر المنشور النهائي |
| **عقد** | Canonical Schema | البنية الموحدة التي يلتزم بها كل مكون |
| **تأثير** | Impact | درجة تأثير الحدث على الأسواق (low/medium/high/critical) |
| **ثقة** | Confidence | مستوى يقين النظام في الحدث (0-100) |
| **موثوقية** | Authority | مستوى رسمية المصدر (official/verified/unverified) |
| **بصمة** | Hash | SHA-256 للتحقق من سلامة الوثيقة |
| **لقطة** | Snapshot | نسخة مؤرشفة من الوثيقة في R2 |
| **دورة حياة** | Lifecycle | حالة الحدث من الاكتشاف للأرشفة |
| **وكيل** | Agent | وحدة AI مستقلة تقوم بدور محدد |
| **مجال** | Domain | تصنيف علوي (macro, corporate, markets, regulatory, geopolitical) |
| **فئة** | Category | تصنيف فرعي داخل Domain (monetary_policy, earnings) |
| **كيان** | Entity | الجهة المرتبطة بالحدث (Federal Reserve, Apple) |
| **أصل** | Asset | أداة مالية متأثرة (USD, GOLD, AAPL, BTC) |
| **اتجاه** | Direction | اتجاه التغير (increase, decrease, neutral) |
| **قيمة** | Value | القيمة الكمية للحدث (+25bps, 3.2%, $1.2B) |
| **نسخة** | Version | نسخة من الوثيقة (للتتبع عند التغيير) |
| **طابور** | Queue | قائمة انتظار للعمليات غير المتزامنة |
| **ميت** | Dead Letter | عملية فشلت نهائياً بعد كل المحاولات |
| **تجميد** | Freeze | إيقاف تطوير مكون حتى إشعار آخر |
| **عقد قرارات** | ADR | Architecture Decision Record |
| **متابعة** | Tracking | لوحة تتبع تقدم المشروع |
| **تراكم** | Backlog | قائمة الميزات المستقبلية |
| **معلم** | Milestone | نقطة فحص رئيسية في المشروع |
